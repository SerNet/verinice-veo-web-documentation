# Dictionary
Some concepts of verinice.veo have different names depending on the context, e.g. depending on whether one interacts with the [user interface](../manual/user-interface), uses the [Web-API](getting-started-with-the-verinice-api.md) or works with the [source code](https://github.com/SerNet?q=verinice-veo).
One possible reason for deviating terms can be that a word already has connotations in a certain technical context (e.g. “object” in [object-oriented programming](https://en.wikipedia.org/wiki/Object-oriented_programming)) and has therefore been replaced with another term.
It is also possible that in the course of the agile development process, a term proves to be unintuitive or not appropriate and is then renamed, but the old term continues to be used in certain software components.
